# 开发文档

## 1. 项目范围

基于现有 Next.js 项目开发一个 AI 生图与 AI 视频生成网站，参考「风暴 AI」的信息架构和交互体验。首版聚焦前台生成流程、历史项目、点数兑换和上游模型接口预留。

参考需求来源：`需求文档.md`

## 2. 技术基础

- 框架：Next.js 16 + React 19 + TypeScript
- 样式：Tailwind CSS 4
- UI：现有 `components/ui/` shadcn 风格组件
- 路径别名：`@/*`
- 启动命令：`pnpm dev`
- 构建命令：`pnpm build`
- 检查命令：`pnpm lint`

## 3. 阶段一：基础界面与导航

### 目标

搭建生图、视频、历史项目和充值入口的基础页面结构。

### 开发内容

- 调整 `app/page.tsx` 为主工作台界面。
- 保留或改造现有 `Sidebar`、`ChatArea`、`ChatInput` 组件。
- 增加顶部或侧边导航：
  - 生图
  - 视频
  - 历史项目
  - 点数充值
- 在首页展示用户点数余额占位。
- 在充值区域放置客服微信二维码占位图。

### 交付物

- 可访问的主界面。
- 四个核心功能入口。
- 移动端和桌面端布局可用。

## 4. 阶段二：生图功能

### 目标

完成生图参数表单和提交流程的前端闭环。

### 开发内容

- 增加提示词输入框。
- 增加图片清晰度选择，例如：
  - 1K
  - 2K
  - 4K
- 增加图片比例选择，例如：
  - 1:1
  - 3:4
  - 4:3
  - 9:16
  - 16:9
- 增加生图模型选择。
- 增加生成按钮、加载状态、错误状态和结果预览区域。
- 暂未接入真实 API 时，使用 mock 数据展示生成结果。

### 交付物

- 生图表单可操作。
- 用户可看到生成中和生成完成状态。
- 生成结果可进入历史项目列表。

## 5. 阶段三：视频生成功能

### 目标

完成视频生成参数表单和提交流程的前端闭环。

### 开发内容

- 增加视频提示词输入框。
- 增加视频时长选择，例如：
  - 5 秒
  - 10 秒
  - 15 秒
- 增加视频清晰度选择，例如：
  - 720p
  - 1080p
- 增加视频模型选择。
- 增加生成按钮、生成中状态、失败状态和视频结果预览。
- 暂未接入真实 API 时，使用 mock 视频卡片展示结果。

### 交付物

- 视频生成表单可操作。
- 用户可看到任务状态。
- 视频生成结果可进入历史项目列表。

## 6. 阶段四：历史项目

### 目标

提供统一的用户历史生成记录。

### 开发内容

- 建立历史项目数据结构：
  - `id`
  - `type`: `image` 或 `video`
  - `prompt`
  - `model`
  - `status`
  - `previewUrl`
  - `createdAt`
- 增加历史项目列表。
- 支持按类型筛选：全部、生图、视频。
- 支持查看生成参数和结果预览。
- 可选支持删除、重新生成、下载。

### 交付物

- 用户可查看生图和视频历史。
- 历史列表与生成流程打通。

## 7. 阶段五：点数与兑换码

### 目标

实现点数余额展示和兑换码兑换流程。

### 开发内容

- 增加点数余额展示。
- 增加兑换码输入框。
- 增加兑换按钮、成功提示和失败提示。
- 建立兑换码接口占位：
  - 校验兑换码。
  - 增加用户点数。
  - 标记兑换码已使用。
- 首页展示客服微信二维码，文案引导用户添加客服购买兑换码。

### 交付物

- 用户可看到点数余额。
- 用户可输入兑换码并获得反馈。
- 客服微信入口清晰可见。

## 8. 阶段六：模型接口对接

### 目标

对接 API Mart 或其他上游中转站，实现真实生图和视频任务。

### 开发内容

- 阅读 API Mart 接口文档，确认：
  - 鉴权方式
  - 生图接口
  - 视频生成接口
  - 任务状态查询
  - 回调机制
  - 错误码
- 在 Next.js 中增加服务端 API Route：
  - `/api/generate/image`
  - `/api/generate/video`
  - `/api/tasks/[id]`
  - `/api/redeem`
- 将上游 API Key 放入 `.env.local`，禁止写入前端代码。
- 统一封装上游请求、错误处理和超时处理。

### 交付物

- 前端生成表单可调用真实服务端接口。
- 服务端接口不泄露上游密钥。
- 任务状态可轮询或通过回调更新。

## 9. 阶段七：账户与数据持久化

### 目标

根据业务需要补齐用户身份、点数、历史项目和兑换码持久化。

### 开发内容

- 确认是否需要登录注册。
- 设计用户表、项目表、兑换码表和点数流水表。
- 接入数据库或后端服务。
- 确保历史项目和点数与用户账户绑定。

### 交付物

- 用户数据可持久保存。
- 点数扣减和兑换记录可追踪。

## 10. 阶段八：验收与上线

### 验收清单

- `pnpm lint` 通过。
- `pnpm build` 通过。
- 生图流程可完整走通。
- 视频流程可完整走通。
- 历史项目展示正确。
- 点数兑换反馈正确。
- 移动端和桌面端主要页面可用。
- API Key 不出现在前端代码和提交记录中。

### 上线前待确认

- 微信二维码正式图片。
- 点数价格和扣减规则。
- 正式模型列表。
- 上游 API 额度、并发限制和失败重试策略。
- 域名、部署平台和环境变量配置。
